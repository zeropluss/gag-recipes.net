## 🎨 图片选择策略

### 主要考虑因素：
1. **视觉吸引力** - 选择色彩丰富、构图美观的图片作为主图
2. **展示完整性** - 优先选择能完整展示菜品特征的角度
3. **用户体验** - 选择最能代表该菜品的经典视角

### 具体选择说明：
- **汉堡**: 选择正面图，更能展示层次感
- **冰淇淋**: 选择短款，更符合经典冰淇淋形象
- **寿司**: 选择拼盘图，展示丰富性
- **蛋糕**: 选择有莓果的版本，更有食欲感
- **披萨**: 选择俯拍图，能看清配料分布
- **沙拉**: 选择有洋葱圈的版本，配菜更丰富
- **热狗**: 选择白盘版本，背景更简洁
- **苹果派**: 选择经典lattice样式
- **三明治**: 选择横拍，更符合查看习惯
- **华夫饼**: 选择方形，更经典
- **甜甜圈**: 选择俯拍，能看到糖霜纹理

## 🔧 技术实现

### 文件结构：
```
gag-recipes.net/
├── image/              # 图片文件夹
│   ├── burger-front.png
│   ├── icecream-short.png
│   └── ...
├── assets/
│   └── recipes.json    # 菜谱数据（已更新图片路径）
└── index.html
```

### 图片引用格式：
```json
{
  "id": "burger",
  "name": "Burger",
  "image": "./image/burger-front.png"
}
```

### 错误处理：
- 如果图片加载失败，会自动降级到 `./images/placeholder.svg`
- 所有图片都设置了 `loading="lazy"` 优化加载性能
- 包含了适当的 `alt` 属性用于无障碍访问
